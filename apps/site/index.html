AI coding harness

길고 복잡한 개발 흐름을 터미널에서 끝까지 이어가는 하네스

SuperLiora는 계획, 조사, 목표 관리, 병렬 실행, 검증, 기억, 문서화, 브라우저/컴퓨터 사용을 하나의 작업 흐름으로 연결합니다. 긴 세션에서도 맥락을 보존하고, 근거 있는 결정을 남기며, 개발자가 다음 단계에 집중할 수 있도록 돕습니다.

Problem

긴 작업은 맥락을 잃고, 결정은 흩어집니다

복잡한 구현, 마이그레이션, 리팩터링은 중간에 목표가 흐려지고, 근거가 사라지며, 같은 질문을 반복하게 됩니다. 도구가 많아질수록 오히려 흐름이 끊깁니다.

Migration & refactoring

대규모 변경에서 계획, 영향 범위, 검증을 하나의 흐름으로 유지합니다.

Feature implementation

요구사항을 명확히 고정하고, 구현 중에도 완료 기준을 잃지 않습니다.

Incident & debugging

조사 근거와 시도한 방법을 정리하며 루트 원인에 집중합니다.

Documentation & knowledge

검증된 결정을 다음 작업에서 다시 사용할 수 있는 자산으로 남깁니다.

Solution

Shift-Tab 한 번으로 Ultrawork 시작

일반 프롬프트는 가볍게, 복잡한 작업은 Ultrawork로 전환합니다. UltraPlan interview부터 UltraGoal, research, Swarm decision, verification, learning까지 이어집니다.

$ |
Ultra workflow

계획부터 검증까지 이어지는 실행 구조

Shift-Tab Ultrawork mode는 복잡한 요청을 UltraPlan interview로 먼저 좁히고, true/false로 검증 가능한 UltraGoal을 만든 뒤 research, Swarm decision, integration, verification 순서로 진행합니다.

Plan → Goal → Research → Swarm → Verify → Learn

각 단계는 이전 단계의 결과를 근거로 시작하며, 완료 여부를 명확히 기록합니다. 모호한 목표는 UltraPlan interview로 구체화되고, 리스크가 큰 결정은 Swarm gate를 통과합니다.

  • UltraPlan interview로 요구사항, 제약, 위험을 끝까지 좁힙니다.
  • UltraGoal로 구체적 목표와 검증 기준을 고정합니다.
  • UltraSwarm에서 ENGAGE/DEFER 결정을 먼저 기록합니다.

Harness capabilities

브라우저와 데스크톱까지 같은 흐름에 담습니다

SuperLiora는 에디터 안에서 끝나지 않습니다. CloakBrowser + Playwright로 웹을 탐색하고, CUA/MCP로 네이티브 데스크톱을 조작하며, provider routing, context, memory, ACP를 같은 하네스에서 다룹니다.

하나의 터미널에서 웹, 데스크톱, 코드 모두 다루기

Browser-use와 computer-use는 별도 도구가 아니라 동일한 harness의 일부입니다. 권한, 기억, 검증 모델을 공유하며 작업 맥락을 유지합니다.

  • CloakBrowser + Playwright로 웹 기반 흐름을 관찰하고 조작합니다.
  • CUA/MCP로 네이티브 창을 캡처하고 클릭, 입력, 드래그를 수행합니다.
  • Provider routing으로 quota, cooldown, latency를 기준으로 fallback합니다.
Memory & LLM Wiki

작업의 맥락을 기록하고 다시 활용합니다

Liora Recall은 프로젝트 사실, 결정, 절차, 후속 작업을 기억합니다. LLM Wiki는 코드베이스와 검증 근거를 문서화해 다음 작업이 같은 출발점을 공유하도록 돕습니다.

  • Semantic memory: 프로젝트 구조, API 제약, 운영 규칙.
  • Procedural memory: 빌드, 배포, 검증 루틴.
  • Governance memory: 정책, 하드 제약, 의사결정 기록.
Premium TUI

터미널도 개발자에게 맞춘 조종석이 됩니다

Neon Noir, Daylight, Prism Syntax, Recall Green, Executive Gold 등 팔레트를 바꾸고 외부 터미널 색상을 가져올 수 있습니다. 키보드 중심 탐색과 세션별 시각 디버깅을 지원합니다.

Neon Operator

기본 다크 팔레트. 높은 대비와 시네마틱한 강조색.

Prism Syntax

코드 중심 작업에 최적화된 구문 강조 팔레트.

Recall Green

장시간 세션에서 눈의 피로를 줄이는 녹색 계열.

Executive Gold

덜 붉은 앰버 계열로 전환한 고급스러운 라이트 팔레트.

UltraPlan

08

quota, cooldown, latency, route health를 기준으로 fallback 후보를 선택합니다.
