길고 복잡한 개발 흐름을 터미널에서 끝까지 이어가는 하네스

SuperLiora는 계획, 조사, 목표 관리, 병렬 실행, 검증, 기억, 문서화, 브라우저/컴퓨터 사용을 하나의 작업 흐름으로 연결합니다. 긴 세션에서도 맥락을 보존하고, 근거 있는 결정을 남기며, 개발자가 다음 단계에 집중할 수 있도록 돕습니다.

Problem

긴 작업은 맥락을 잃고, 결정은 흩어집니다

복잡한 구현, 마이그레이션, 리팩터링은 중간에 목표가 흐려지고, 근거가 사라지며, 같은 질문을 반복하게 됩니다. 도구가 많아질수록 오히려 흐름이 끊깁니다.

Migration & refactoring

대규모 변경에서 계획, 영향 범위, 검증을 하나의 흐름으로 유지합니다.

Feature implementation

요구사항을 명확히 고정하고, 구현 중에도 완료 기준을 잃지 않습니다.

Incident & debugging

조사 근거와 시도한 방법을 정리하며 루트 원인에 집중합니다.

Documentation & knowledge

검증된 결정을 다음 작업에서 다시 사용할 수 있는 자산으로 남깁니다.

Solution

Shift-Tab 한 번으로 Ultrawork 시작

일반 프롬프트는 가볍게, 복잡한 작업은 Ultrawork로 전환합니다. UltraPlan interview부터 UltraGoal, research, Swarm decision, verification, learning까지 이어집니다.

계획부터 검증까지 이어지는 실행 구조

Shift-Tab Ultrawork mode는 복잡한 요청을 UltraPlan interview로 먼저 좁히고, true/false로 검증 가능한 UltraGoal을 만든 뒤 research, Swarm decision, integration, verification 순서로 진행합니다.

Harness capabilities

브라우저와 데스크톱까지 같은 흐름에 담습니다

SuperLiora는 에디터 안에서 끝나지 않습니다. CloakBrowser + Playwright로 웹을 탐색하고, CUA/MCP로 네이티브 데스크톱을 조작하며, provider routing, context, memory, ACP를 같은 하네스에서 다룹니다.

Browser-use CloakBrowser + Playwright

observe, act, screenshot, console을 하나의 흐름에서 사용해 웹 기반 작업을 검증합니다.

Computer-use CUA / MCP

네이티브 창을 캡처하고 클릭, 입력, 스크롤, 드래그까지 수행해 GUI 디버깅과 자동화를 이어갑니다.

Provider routing quota, cooldown, latency

여러 API key와 OAuth account를 등록하고 상태를 기준으로 fallback 후보를 선택합니다.

Memory and LLM Wiki

작업의 맥락을 기록하고 다시 활용합니다

Liora Recall은 프로젝트 사실, 결정, 절차, 후속 작업을 기억합니다. LLM Wiki는 코드베이스와 검증 근거를 문서화해 다음 작업이 같은 출발점을 공유하도록 돕습니다.

Semantic memory facts

프로젝트 구조, API 제약, 팀 규칙을 보존합니다.

Procedural memory how-to

반복되는 빌드, 배포, 검증 절차를 다시 사용할 수 있게 정리합니다.

Governance memory rules

중요한 정책과 금지 사항을 이후 작업에서도 확인할 수 있게 유지합니다.

Premium operator surface

오래 머무는 터미널을 더 읽기 쉽게

긴 세션에서는 시각 피로, 코드 가독성, 상태 인지가 작업 품질에 영향을 줍니다. SuperLiora는 preset, 외부 terminal palette, syntax-aware color를 정제된 TUI 경험으로 제공합니다.

Neon Operator

명령, 상태, diff를 선명하게 구분합니다.

Prism Syntax

긴 코드 블록과 reasoning 로그의 가독성을 높입니다.

Recall Green

memory, wiki, evidence 흐름을 쉽게 구분합니다.

Executive Gold

review, verify, release gate에 필요한 대비를 제공합니다.

Premium capabilities

현장에서 필요한 기능을 한 흐름에 담았습니다

SuperLiora는 계정과 API, context, research, memory, theme, workflow를 따로 관리하지 않고 같은 에이전트 흐름에서 다룹니다.

01

UltraPlan

목표가 true/false로 검증 가능해질 때까지 요구사항과 제약을 인터뷰합니다.

02

UltraResearch

API, 논문, 릴리스 노트, 보안 이슈를 확인하고 근거를 남깁니다.

03

UltraGoal

목표, 예산, 완료 조건을 명확히 두어 장기 작업의 방향을 유지합니다.

04

UltraSwarm

최대 128개의 specialist subagent를 ENGAGE/DEFER gate 뒤에서 투입합니다.

05

Context OS

긴 세션을 structured working memory, repair, bounded rehydration으로 관리합니다.

06

Liora Recall

프로젝트 사실, 결정, 절차, governance rule을 세션 간에 보존합니다.

07

LLM Wiki

코드베이스 지식, 조사 근거, 검증 결과를 프로젝트 로컬 wiki로 정리합니다.

08

Browser-use

CloakBrowser + Playwright로 웹 페이지를 탐색하고 검증합니다.

09

Computer-use

10

Provider routing

quota, cooldown, latency, route health를 기준으로 fallback 후보를 선택합니다.

11

Premium TUI

Neon Noir / Daylight 팔레트와 ambient effect로 긴 터미널 세션을 돕습니다.

12

ACP ready

터미널과 호환 editor에서 같은 SuperLiora workflow를 이어갈 수 있습니다.

13

Visual debugging

세션, replay, context, trace를 시각적으로 추적하고 검토합니다.

14

Server API

REST와 WebSocket(`/api/v1`)으로 세션을 노출해 클라이언트와 자동화가 연결합니다.

15

Non-interactive

TUI에 들어가지 않고 `liora -p`로 명령을 실행하고 CI/CD에 연결합니다.

Visual debugging

세션과 결정을 눈으로 추적합니다

`apps/vis`는 에이전트 실행의 흔적을 세션, replay, context, trace 단위로 시각화합니다. 긴 작업이 어디서 꺾였는지, 어떤 근거가 쌓였는지 확인할 수 있습니다.

Session list

실행 중이거나 저장된 세션을 한눈에 보고, 원하는 시점으로 재개합니다.

Trace replay

턴과 도구 호출, 결정의 흐름을 다시 재생해 루트 원인을 찾습니다.

Context projection

Context OS가 선택한 메모리 페이지와 연속성 상태를 시각적으로 확인합니다.

Install

소스에서 설치하고 liora로 시작하세요

설치 스크립트는 SuperLiora를 독립 명령으로 빌드하고 실행 경로를 설정합니다. macOS/Linux는 curl, Windows는 PowerShell one-liner로 시작할 수 있습니다.

$ curl -fsSL https://raw.githubusercontent.com/claudianus/superliora/main/install.sh | bash
PS> irm https://raw.githubusercontent.com/claudianus/superliora/main/install.ps1 | iex
$ liora

Shift-Tab로 Ultrawork를 시작해 보세요
