Ultra workflow

Execution from plan to verification

Shift-Tab Ultrawork mode gathers source-backed research first, narrows complex requests through UltraPlan interview, creates a true/false-verifiable UltraGoal, then proceeds through Swarm decision, integration, and verification.

Plan → Goal → Research → Swarm → Verify → Learn

Each step starts from the previous step's output and records clear completion criteria. Fuzzy goals are sharpened by the UltraPlan interview; high-risk decisions pass through the Swarm gate.

  • UltraPlan interview narrows requirements, constraints, and risks.
  • UltraGoal pins the concrete objective and verification criteria.
  • UltraSwarm records ENGAGE/DEFER before any specialist runs.

Harness capabilities

Beyond the editor: browser, desktop, and routing

SuperLiora does not stop at the editor. It drives CloakBrowser + Playwright for the web, CUA/MCP for native desktop, and routes providers by quota, cooldown, latency, and health — all inside the same harness.

Web, desktop, and code in one terminal

Browser-use and computer-use are not separate tools — they are part of the same harness. They share permission, memory, and verification models so context stays intact.

  • CloakBrowser + Playwright observes and acts on web flows.
  • CUA/MCP captures native windows and performs click, type, drag.
  • Provider routing falls back by quota, cooldown, and latency.
Memory & LLM Wiki

Record project context and use it again

Liora Recall preserves project facts, decisions, procedures, and follow-up work. LLM Wiki turns codebase knowledge and verified evidence into documentation the next task can share.

  • Semantic memory: project structure, API constraints, operating rules.
  • Procedural memory: build, deploy, and verification routines.
  • Governance memory: policies, hard constraints, decision records.
